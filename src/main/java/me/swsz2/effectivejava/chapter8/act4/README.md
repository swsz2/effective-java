# 다중정의(Overloading)는 신중히 사용하라

## Overloading
    - 두 메서드가 다른 이름을 갖고 있으나 인자의 수나 자료형이 다른 경우

## 오버로딩에 신중해야 하는 이유
    - 오버로딩된 메소드 중 어느 메소드를 호출할지는  컴파일 타임에 정해진다.
        - 재정의한 메소드(@Overide)는 동적으로 선택되고, 다중 정의(Overide)한 메소드는 정적으로 선택된다.
        -> 작성 의도와 다르게 동작할 수 있다.
    - 사용자에게 혼동을 줄 수 있다.

## 김성원의 생각
크게 느낀 점이 없다. <br>
오버로딩 자체를 사용하지 않아서 그런 거 같다. <br>
내가 오버로딩을 사용하지 않는 이유는 '어떤 함수의 동작은 메소드 이름에서 명확히 알 수 있어야 한다.'라는 생각이 있어서다. <br>
어떠한 메소드를 오버로딩해서 여러 개 구현하게 되면 사용자는 매개변수와 메소드명을 가지고 메소드의 동작을 **유추**해야 한다. <br>
메소드의 동작을 유추해야 한다는 것 자체가 신뢰도가 낮은 메소드라는 뜻이다. <br>
결국 신뢰도가 낮은 메소드는 클라이언트가 사용하기 어렵다. <br>
그래서 애초에 메소드명으로 동작을 명확히 안내하는 것을 선호한다. <br>
이것이 내가 오버로딩을 사용하지 않는 이유이다.
